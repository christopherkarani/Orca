# Customer Safety Reports

`safety-report.md` is intended for customer evaluation demos. It is concise, readable, and includes tables for commands, safety findings, evidence, and traceability.

Every customer report must include:

- policy hash
- scenario id and session id
- fake/PX4 SITL/ArduPilot SITL/bench/unknown provenance
- exact commands and decisions
- safety findings
- audit hash-chain status
- limitations
- non-certification disclaimer

Customer reports must not claim real-flight validation, regulatory approval, certification, detect-and-avoid coverage, autopilot replacement behavior, or hardware deployment readiness.
