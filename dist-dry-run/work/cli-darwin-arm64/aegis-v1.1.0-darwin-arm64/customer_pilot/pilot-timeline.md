# Pilot Timeline

## Week 0 / Pre-pilot

- Discovery call.
- Customer stack mapping.
- Policy scope selection.
- Evaluation environment selection: fake adapter, PX4 SITL, ArduPilot SITL, or bench-preparation/no-actuation.
- Success criteria agreement.
- Safety boundary review.

## Week 1

- Local setup.
- Fake/SITL integration check.
- Baseline policy creation.
- Initial scenario runs.
- Audit/replay verification.
- Early limitations log.

## Week 2

- Red-team and fault-injection run.
- Safety-case report generation.
- Data guard and runtime health review.
- Findings review.
- Next-step recommendation.

## Optional Week 3-4

- Deeper customer-specific scenarios.
- SITL expansion.
- Bench-preparation review with no actuation.
- Policy refinement.

No timeline step includes real flight or live aircraft control.
