# Target Account Template

Use public company information, warm introductions, and manually supplied business contact information only. Do not scrape private data or add personal contact details that were not intentionally shared for business outreach.

| Field | Notes |
| --- | --- |
| company | Company name. |
| segment | Inspection, warehouse, agriculture, public safety, fleet, robotics, or other. |
| website | Public website. |
| contact name | Business contact if known. |
| contact role | CTO, Head of Autonomy, Robotics Lead, Safety Lead, Platform Lead, Founding Engineer, VP Engineering. |
| contact source | Public site, event page, warm intro, direct reply, or user-supplied context. |
| warm intro source | Person who can introduce, if any. |
| autopilot/stack guess | Publicly visible stack hypothesis only. |
| PX4/ArduPilot/MAVLink evidence | Link or note from public source. |
| simulation/SITL evidence | Link or note from public source. |
| likely pain | Safety evidence, audit/replay, command mediation, red-team, customer proof, or policy baseline. |
| outreach status | Identified, researched, contacted, replied, booked, proposed, disqualified. |
| last touch | Date of last manual touch. |
| next step | Specific next action. |
| call booked | Yes/no/date. |
| pilot interest | None, low, medium, high. |
| notes | Relevant business notes only. |
| disqualifying risks | No SITL, unsafe market, approval claim expectation, no technical discovery. |
| safety boundary concerns | Any sign the account expects real-flight evidence, certification, or aircraft control. |
