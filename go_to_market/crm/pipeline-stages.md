# Pipeline Stages

1. identified
2. researched
3. contacted
4. replied
5. discovery booked
6. demo completed
7. technical validation
8. pilot proposed
9. pilot accepted
10. pilot started
11. pilot completed
12. closed lost
13. disqualified

## Exit Criteria

- Identified -> researched: public stack and segment notes added.
- Researched -> contacted: manual founder-led message sent.
- Contacted -> replied: prospect responded.
- Replied -> discovery booked: calendar time agreed.
- Discovery booked -> demo completed: demo run or proof bundle shown.
- Demo completed -> technical validation: stack fit and simulator path likely.
- Technical validation -> pilot proposed: one workflow, owners, and success criteria identified.
- Pilot proposed -> pilot accepted: customer agrees to scope and commercial terms.
- Pilot accepted -> pilot started: kickoff complete.
- Pilot started -> pilot completed: report delivered and reviewed.
- Any stage -> closed lost: not proceeding but safe fit.
- Any stage -> disqualified: safety, scope, or fit boundary failed.
